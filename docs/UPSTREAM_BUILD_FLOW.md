# Upstream Build Flow

## Official Entry Point

**`buildiso.sh`** — invoked as `bash buildiso.sh -p desktop -v`

This is the single entry point intended by CachyOS. It is NOT a wrapper around mkarchiso — it is the build orchestrator. All profile preparation, file generation, and build execution is handled internally. The workflow must call this script directly, never mkarchiso.

## Script Execution Order

```
buildiso.sh
├── Sources: util-msg.sh        (msg/info/warning/error output)
├── Sources: util.sh            (helper functions: timer, check_root, check_requirements, etc.)
├── Parses CLI args (-p profile, -v verbose, -c clean, -r ram, -w remove)
├── Sources: util-iso.sh        (core build logic: prepare_profile, run_build)
├── Sources: util-iso-mount.sh  (mount helpers for chroot)
└── Calls: run_build "${profile}"
    └── util-iso.sh:run_build()
        ├── prepare_profile("desktop")
        │   ├── change_grub_version()        — patches archiso/grub/grub.cfg version string
        │   ├── fetch_cachyos_mirrorlist()   — downloads mirrorlist from GitHub
        │   ├── generate_motd()              — writes archiso/airootfs/etc/motd
        │   ├── cp packages_desktop.x86_64   → packages.x86_64
        │   ├── ln -s plasmalogin.service    → display-manager.service
        │   ├── generate_environment()       — writes archiso/airootfs/etc/environment
        │   ├── generate_version_tag()       — writes archiso/airootfs/etc/version-tag
        │   ├── generate_edition_tag()       — writes archiso/airootfs/etc/edition-tag
        │   └── gen_iso_fn()                 — sets iso_file variable
        ├── (optional) rm -rf ${work_dir}    — clean build directory
        ├── mkdir -p ${work_dir}
        ├── cp -r archiso ${work_dir}/archiso/   — copies prepared profile → work dir
        ├── modify_mkarchiso()               — patches /usr/bin/mkarchiso (remove keyring timer)
        ├── sudo mkarchiso -v -w ${work_dir} -o out/${profile} ${work_dir}/archiso/
        │   └── mkarchiso (system tool from archiso package)
        │       ├── Validates profile (packages, bootmodes, pacman.conf, etc.)
        │       ├── Bootstrap pacstrap rootfs
        │       ├── Install packages from merged package list
        │       ├── Run customize_airootfs hooks
        │       ├── Create squashfs filesystem
        │       ├── Build bootable ISO (GRUB, syslinux, kernel, initramfs)
        │       └── Write ISO file to output directory
        ├── cp pkglist.x86_64.txt → out/${profile}/
        ├── mv ISO → out/${profile}/${iso_file}  (renames to profile-specific name)
        ├── create_chksums()  — sha1 + sha256
        ├── sign_with_key()   — GPG sign if key configured
        └── (optional) rm -rf ${work_dir}
```

## Generated Intermediate Files

The following files are created or modified **in-place** within the source tree during `prepare_profile()`. They are NOT permanent — a fresh checkout restores the originals.

| File | Source | Purpose |
|------|--------|---------|
| `archiso/packages.x86_64` | Copied from `packages_desktop.x86_64` | Default package list expected by mkarchiso |
| `archiso/airootfs/etc/motd` | Generated by `generate_motd()` | Login message |
| `archiso/airootfs/etc/environment` | Generated by `generate_environment()` | Systemd environment variables |
| `archiso/airootfs/etc/version-tag` | Generated by `generate_version_tag()` | ISO build date version |
| `archiso/airootfs/etc/edition-tag` | Generated by `generate_edition_tag()` | Profile name (e.g., "desktop") |
| `archiso/airootfs/etc/pacman.d/cachyos-mirrorlist` | Fetched from GitHub | CachyOS repository mirrorlist |
| `archiso/grub/grub.cfg` | Modified by `change_grub_version()` | Version string injected into GRUB boot menu |
| `archiso/airootfs/etc/systemd/system/display-manager.service` | Symlink to plasmalogin.service | SDDM/Plasma login manager |

During build, a **copy** of the entire modified `archiso/` directory is made to `${work_dir}/archiso/`. All mkarchiso operations run on this copy.

## Build Output Files

All output goes to `out/${profile}/`:

| File | Description |
|------|-------------|
| `cachyos-${profile}-linux-${YYMMDD}.iso` | Bootable ISO image |
| `cachyos-${profile}-linux-${YYMMDD}.iso.sha1` | SHA-1 checksum |
| `cachyos-${profile}-linux-${YYMMDD}.iso.sha256` | SHA-256 checksum |
| `cachyos-${profile}-linux-${YYMMDD}.iso.sig` | GPG signature (if key configured) |
| `${iso_file}.pkgs.txt` | Package list snapshot from the built ISO |
| `poc-report.txt` | (Alya PoC only) Build metrics report |

The final ISO filename is set by `gen_iso_fn()` which constructs:
`cachyos-${profile}-linux-${YYMMDD}`

## Permanent Files vs Generated Files

### Permanent (tracked in repo — must exist before build)
- `buildiso.sh` — entry point
- `util-iso.sh` — build orchestration (prepare_profile, run_build)
- `util-iso-mount.sh` — chroot mount helpers
- `util.sh` — generic utility functions
- `util-msg.sh` — formatted messaging
- `archiso/packages_desktop.x86_64` — primary package list (never `packages.x86_64`)
- `archiso/profiledef.sh` — mkarchiso profile definition
- `archiso/pacman.conf` — pacman configuration for the ISO
- `archiso/grub/` — GRUB bootloader configuration
- `archiso/syslinux/` — syslinux bootloader configuration
- `archiso/efiboot/` — UEFI boot configuration
- `archiso/airootfs/` — root filesystem overlay (excluding generated files above)

### Generated (not tracked — created during build)
- `archiso/packages.x86_64` — copied from `packages_desktop.x86_64`
- `archiso/airootfs/etc/motd` — runtime generated
- `archiso/airootfs/etc/environment` — runtime generated
- `archiso/airootfs/etc/version-tag` — runtime generated
- `archiso/airootfs/etc/edition-tag` — runtime generated
- `archiso/airootfs/etc/pacman.d/cachyos-mirrorlist` — fetched from GitHub
- `archiso/airootfs/etc/systemd/system/display-manager.service` — symlink
- `archiso/grub/grub.cfg` — modified in-place (restored by fresh checkout)
- `build/` — work directory (intermediate; can be deleted)
- `out/` — output directory (final artifacts)

## Where mkarchiso Is Actually Called

`util-iso.sh:187` — inside `run_build()`:

```bash
sudo mkarchiso -v -w ${work_dir} -o "$outFolder/$_profile" ${work_dir}/archiso/
```

Arguments:
- `-v` — verbose (only if `-v` passed to buildiso.sh)
- `-w ${work_dir}` — work directory (defaults to `build/`)
- `-o out/${profile}` — output directory
- `${work_dir}/archiso/` — the **copied**, **prepared** profile directory

Note: The profile passed to mkarchiso is `${work_dir}/archiso/` (a copy), NOT the source tree. This is important because mkarchiso modifies certain files during validation and would otherwise dirty the source tree.

## Key Implications for CI

1. **Call `buildiso.sh`, never `mkarchiso` directly.** All prep work (package list copy, motd, mirrorlist, version tags, display manager) is done by `prepare_profile()` before mkarchiso runs. Skipping it causes the exact failure we saw: `packages.x86_64` missing.

2. **`packages.x86_64` does not exist in the repository.** Only `packages_desktop.x86_64` exists. The copy is intentional — CachyOS supports multiple profiles (desktop, minimal, gaming) each with their own package list. mkarchiso always expects `packages.{arch}`.

3. **sudo is required.** The upstream script uses `sudo mkarchiso` and `sudo sed -i /usr/bin/mkarchiso`. The CI container must have sudo installed and running as a privileged user.

4. **The source tree is modified during build.** `prepare_profile()` writes files in-place. This is expected — `git checkout .` restores the originals. For CI, this is irrelevant since the clone is fresh per run.

5. **Network access is required.** `fetch_cachyos_mirrorlist()` downloads from GitHub during build. This is normal for CachyOS — the mirrorlist is updated constantly and not tracked in the repo.

6. **`/usr/bin/mkarchiso` is patched at runtime.** `modify_mkarchiso()` removes the `archlinux-keyring-wkd-sync.timer` from the build to avoid timer activation during live session. This is ephemeral in the container.
